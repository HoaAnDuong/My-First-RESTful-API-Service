1. SQLite Database

In this project, I used SQLite3 to build a Database. SQLite3 is a tool help programmers implement a small, fast, self-contained, high-reliability, full featured SQL Database. Despite its benefit, SQLite has many problems in data structure, such as not have a proper type for date storage, cannot change column type normally (Due to that, I have Recreate Entire Database twice), .etc. However, SQLite is still good for small-scale Database. 

To connect and excute query on Database, I used  SQLite command-line shell  program

Python have a SQLite-supported module called 'sqlite3'. In 'sqlite3', you can spawn cursors for executing SQLIte statements.

The Database have a total of 10 tables containing all data for Store Management:

  • Customers: contains Customers's information

  • Shippers: contains Shippers's information 
  • Suppliers: contains Suppliers's information 
  • Categories: contains Categories's information 
  • Products: contains Products's information 
  • Orders: contains Orders's information 
  • OrderDates: contains Orders's date
  • OrderPayments: contains Orders payment's information

2.2. Sending Requests

Before going to Sending Requests, we must find out what a HTTP is.

"The Hypertext Transfer Protocol (HTTP) is designed to enable communications between clients and servers.

HTTP works as  a request-response protocol  between a client and server.

Example: A client (browser)  sends an HTTP request  to the server; then the server  returns a response  to the client. The response contains status information about the request and may also contain the requested content."

"HTTP Request Methods "-W3Schools

There are 9 HTTP Methods(Requests): GET, POST, PUT, HEAD, DELETE, PATCH, OPTION, CONNECT, TRANCE. Despite its variety, the action based on these Methods are actually decided by Back-end Developers (for example: The server can update a row in a database instead of delete it when received a DELETE Request) 

In this website, you can get, change and delete data of a Store's Database via Sending Requests. There are 4 Request Methods corresponding to 4 Actions on Store's Database:

  • GET (SELECT): Get datas from a Table(or a Row)
  • POST (INSERT): Insert data into a Table
  • PUT(UPDATE): Update data on a Row
  • DELETE(DELETE): Delete data on a Row

3.1 Deploying SQLite Database

After read the given task, I decided to deploy an SQLite Datbase using SQLite command-line shell program. It took me ~2 days to Deploy a Database like below image.

But a problem came up: how to store Order Date and Total? Did I just need to add more columns like Date, PaymentMethod, Total, Status in Orders? But after doing that, The Orders table just liked a mess. So, I decided to store Date and Payment in two seperated Table. 

I thought everything would be fine, but more and more problems came up: How to calculate the Order Total? How to prevent illegal changes on Orders and its Relatives? After a lesson of Database in school, my teacher recommended using triggers to enforce rules on the Database. I tried to search infomations about SQLite Trigger. At first, I tried writing some small triggers like auto add date and payment, block update on OrderDate, etc. After many day, I wrote a lot of triggers, enforced a lot of rules on the Database, make things worked automatically.

However, another big problem came up: I forgot to add primary key to some table and make them unique, so I need to backup and recreate entire Database following these step:

0. pragma foreign_keys = off;

1. Begin transaction;

2. Backup all the SQL query belong to tables and triggers (select sql from sqlite_master where type = 'table')(select sql from sqlite_master where type = 'trigger' )

3. Drop all triggers, rename table to table_olds

4. Fix the backup SQL query 

5. Recreate all the tables and triggers using backup SQL query

6. Insert into table select * from table_olds (Copy data from table_olds)

7. Drop all table_olds

8. Commit;

9. pragma foreign_keys = on;
